解説と回答例

この課題のねらい

この課題のねらいは、「要件定義 → AIにJSONを書かせる → インポート → 調整 → 実行」という、n8n実装の最短ルートを体験することです。

従来の業務自動化は、ノードを1つずつ手で並べて組み立てるのが当たり前でした。しかしAIが成熟した今は、「要件さえ明確ならコード（JSON）は書いてもらえる」時代です。人間の役割は、要件を磨くこと・生成物を読んで取捨選択すること・自分の業務に合わせて調整すること、の3つに絞られます。

JSONとは何か（ざっくり）

JSONは「システム同士が使う帳票フォーマット」のようなものです。`{` `}` で囲まれたブロックの中に「キー：値」の組が並んでいる形式で、機械にも人にも読めるように設計されています。

{ "name": "現在時刻をセット", "type": "n8n-nodes-base.set", "parameters": { "values": { "hour": "={{ $now.hour }}" } } }

上の例は「"現在時刻をセット" という名前のSetノードで、hour という変数に今の時刻を入れる」という設定を表しています。中身の意味を一字一句理解する必要はありません。「何となくどのノードのことを言っているか分かる」レベルで十分です。

AIにJSONを書かせるコツ

AIに「n8nのJSONを書いて」とだけ頼むと、外部サービス（Gmail / Slack など）をそのまま使うJSONを出してくることが多い。この課題では外部接続を避けたいので、プロンプトに以下を明示するのがポイントです。

  • 「Manual Trigger にしてください」で起動方法を固定する
  • 「外部サービス接続が必要な部分は Edit Fields (Set) でダミーデータを返す擬似ノードに置き換えて」と明示する
  • 「n8nのインポート欄に貼れるJSON全文のみ出力、解説文は不要」と出力形式を指定する
  • 長すぎるJSONが返ってきたら 「3〜5ノードにシンプル化して」 と差し戻す

回答例：採用市場ニュース要約フローを擬似化して実装

AIに投げたプロンプト

以下の要件定義書に沿って、n8nでそのままインポートして動かせるワークフローのJSONを書いてください。 【条件】 ・トリガーは Manual Trigger にしてください ・外部サービス接続が必要な部分（RSS取得／Gemini／Slack）は、 　Edit Fields (Set) でダミーデータを返す擬似ノードに置き換えてください ・各ノードには日本語で分かりやすい名前を付けてください ・n8nのインポート欄に貼れるJSON全文のみを出力してください（解説文は不要） 【要件定義書】 （セクション1で作成した要件定義書の全文）

AIが生成したJSONの骨格

1. Manual Trigger（手動起動） 2. Set: ダミー記事データ（タイトル3件、概要3件）をセット 3. Set: ダミーAI要約（3行の要約文）をセット 4. If: 「採用市場 or HRテック」キーワードが含まれるか判定 5. Set: Slack投稿用メッセージを整形（実際には投稿せずメッセージ文だけ作る）

パラメータ調整のBefore/After

調整箇所1： 　ノード名：Set_ダミー記事データ 　Before：ダミー記事A／ダミー記事B／ダミー記事C 　After ：採用市場の求人倍率が2.8倍に上昇／SmartHRがAI分析機能追加／ テレワーク推進ガイドライン改定 調整箇所2： 　ノード名：If_キーワード判定 　Before：条件「title contains "dummy"」 　After ：条件「title contains "採用" or title contains "HR"」

調整後に実行すると、If分岐で「採用市場」関連の記事だけが次のノードに流れ、Slack投稿用メッセージが整形されて完成。Executionsに「Success」が記録されます。

よくあるつまずきどころ

  • JSONをインポートしたらエラーになった：AIが出力したJSONに不要なコードブロック記号（` ``` `）が混ざっていることが多い。先頭と末尾の余計な記号を削ってから貼り直す。
  • インポートはできたが実行で赤エラー：AIが既存ワークフローの参照（Node IDなど）を残していることがある。エラーメッセージに従ってIDをリセットするか、該当ノードを削除して作り直す。
  • 思った通りに分岐しない：If/Switchの条件式にAIが不正確な参照を書いていることがある。手で条件を設定し直すのが早い。
  • ノードが多すぎて全体が把握できない：AIは時々過剰にノードを作る。不要なものは削除して、自分が理解できる規模（5〜7ノード）に絞ると学習効果が高い。

この方法の価値と注意点

価値：要件定義ができていれば、実装時間は数時間 → 数十分に短縮できる。しかも「AIに書かせる → 読んで直す」の繰り返しで、徐々にJSONやノード構成の理解も進む。

注意点：AIの生成したJSONを何も読まずにそのまま動かすのは危険。後でバグが出ても原因を追えなくなる。必ず自分で中身を理解できるレベルまで削る・書き換える意識を持つこと。

実務での活用イメージ

  • S4で外部APIと繋ぐ：今回は擬似Setノードで済ませた外部接続を、セクション4で本物のGmail／Slack／Gemini等に置き換えます。すでにフロー全体が動いている状態から外部接続を差し込むだけなので、スムーズに進みます。
  • 業務の自動化提案：要件定義書＋AIが書いたJSON＋簡易デモが30分で揃うので、上司やチームに「こういう自動化が可能です」と提案しやすくなる。
  • テンプレート化：1つうまく動いたフローを保存しておけば、似た業務を自動化するときの出発点として使い回せる。
