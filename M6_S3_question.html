実習課題：要件定義書からAIにn8nフローを書かせ、動かしてみよう

目的

セクション1で作った要件定義書を、いよいよ実際のn8nワークフローに変えていきます。ゼロからノードを並べて組むのではなく、AIにn8n用のJSON（設計図）を書いてもらい、それをインポートして動かすという方法を体験します。

この方法のポイントは、「作業の大部分をAIに任せ、人間は要件の明確化とパラメータ調整に集中する」こと。要件定義さえしっかりしていれば、実装の時間は数十分に短縮できます。

この課題の目的は、①AIにJSONを書かせる／②インポートしてキャンバスで中身を確認／③自分の業務に合わせてパラメータを微調整／④実行成功という一連の流れを、自分の手で1周することです。

課題

セクション1で作成した自分の要件定義書をベースに、AIにn8n用のワークフローJSONを生成させてください。それをn8nにインポートし、必要な箇所のパラメータを自分の業務に合わせて調整したうえで、実行成功まで持っていってください。

外部サービス接続はまだしなくてOK
Gmail／Slack／Geminiなど外部APIとの接続は、セクション4で本格的に扱います。セクション3では、外部接続が必要な部分を内蔵ノード（Edit Fields / Set）で擬似的に置き換えて実行してください。AIへの依頼文にもその旨を書きます（後述のプロンプト例参照）。

成果物

  • AIに投げたプロンプト ── 要件定義書＋JSON生成依頼の全文（テキスト）
  • AIが生成したJSON ── コピー可能な形で貼り付け（長ければ先頭〜末尾を抜粋＋行数記載でも可）
  • パラメータ調整のBefore/After ── 1〜2箇所、どこを何に変えたかを記述（テキスト）
  • 実行成功時のワークフロー画面スクリーンショット ── インポートして調整したワークフローを実行し、すべてのノードが緑色のチェックで表示されている（=正常実行）状態の1画像
  • 振り返り ── 3〜5行の短い感想（何が便利だったか／どこでつまずいたか）

手順

ステップ① AI（ChatGPT／Gemini）にJSONを書かせる

セクション1で作った要件定義書を貼り付けて、以下のようなプロンプトでJSONを生成してもらいます。

AIへのプロンプトの例

以下の要件定義書に沿って、n8nでそのままインポートして動かせるワークフローのJSONを書いてください。

【条件】
・トリガーは Manual Trigger にしてください（手動で「Execute workflow」ボタンを押すと動く形）
・外部サービス接続が必要な部分（Gmail / Slack / Gemini API など）は、今回は実際に接続しません。代わりにEdit Fields (Set) ノードでダミーデータを返す擬似ノードに置き換えてください
・分岐がある場合は If または Switch ノードで実装してください
・各ノードには日本語で分かりやすい名前を付けてください
・最後に、n8nのインポート欄に貼れるJSON全文のみを出力してください（解説文は不要）

【要件定義書】
（ここにセクション1で作った要件定義書の全文を貼り付け）
うまくJSONが生成されないときは
  • AIの出力が途中で切れた → 「続けて」と伝える。または「もっとシンプルに、3〜5ノードに収めて」と指示
  • JSONの形式が壊れている → 「n8nにそのままインポートできる有効なJSON形式で出し直して」と差し戻す
  • AIが外部サービスをそのまま使おうとする → 「SlackやGmailなどのノードは使わず、Edit Fields (Set) で置き換えて」と再指示

ステップ② n8nにインポートする

  1. n8nで新規ワークフローを作成（画面の「+」ボタン／「Start from scratch」など）
  2. AIが生成したJSONをテキストファイルとして保存する（拡張子は .json を推奨）
  3. キャンバス右上の「⋯」メニューを開き、「Import from file...」を選択
  4. ステップ2で保存したJSONファイルを選んで読み込む
  5. キャンバスにノードが並び、線でつながっていることを確認
  6. ワークフローに分かりやすい名前を付けて「Save」ボタンで保存

ステップ③ パラメータを自分の業務に合わせて調整する

AIが書いたJSONは、あくまで要件定義の"たたき台"です。キャンバス上のノードを1つずつクリックして中身を見て、自分の業務に合うように1〜2箇所パラメータを書き換えてください。

書き換えの候補例
  • 擬似Setノードのダミー値を、実際の業務でありえそうな具体的なデータに書き換える（例：ダミーの記事タイトル → 自社プロダクトの事例タイトル）
  • If / Switch の分岐条件を、自分の業務の実際のルールに合わせる（例：「金額 > 10000 なら要承認」）
  • メッセージ文言や件名を、実際のチームで使う表現に書き換える
  • Set ノードでセットする変数名を、自分の用語に合わせる
必ず調整前（Before）と調整後（After）の値を控えてください（提出に使います）。

ステップ④ 実行して「Success」にする

  1. キャンバス下部中央の「Execute workflow」ボタンをクリック
  2. すべてのノードに緑のチェックが付き、実行成功時に右下に「Workflow executed successfully」の通知が出ることを確認
  3. エラーで止まった場合は、赤くなったノードのメッセージを確認して修正、または AI にエラー内容を貼って相談（「JSON先頭の ``` 記号を消す」で解決することが多い）
  4. 再度実行し、全ノードが緑になった状態でスクリーンショットを撮影

提出物
