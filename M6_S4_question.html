実習課題：外部APIとAIを組み合わせた情報収集＋要約フローを作ろう

目的

セクション3では、外部サービスに接続する部分をSetノードで擬似化して動かしました。セクション4では、いよいよ本物の外部APIと通信するフローを作ります。

この課題の目的は3つあります。

  1. HTTP Requestノードで本物のAPIを叩き、実データを取得すること
  2. n8nのCredentials機能でAPIキーを安全に管理すること（ハードコード禁止）
  3. 取得したデータをAI（Gemini／OpenAI）で処理し、人が読める結果に変換すること

小さくても、「外部の世界からデータを取ってきて・AIで加工して・出力する」というAIエージェントの基本構造を1周体験するのがゴールです。

課題

以下の要素をすべて含むn8nワークフローを作成し、実行成功まで持っていってください。

  • 起動トリガー：Manual Trigger（Schedule Triggerも可）
  • 外部API呼び出し：HTTP Requestノードで実APIを1回以上叩く
  • AI処理：Gemini または OpenAI ノードでデータを要約／整形／判定など
  • 認証管理：APIキーはn8nのCredentials機能に保存し、HTTP Requestノードから参照する（URL・Headerに直書きNG）
  • 実行成功：Executionsに「Success」として記録されていること
代替オプション：Webhook受信型でもOK
「外から情報を取りに行く」のではなく、「外から情報を受け取る」パターン（Webhook）で作ってもOKです。その場合、Webhookノードで受けて → AIで処理 → 結果を返す／保存する、という構成になります。Webhookのテストは n8n の「Test URL」にcurlやSlackのOutgoing Webhookから送信して動作確認します。

成果物

  • 使用したAPIの説明 ── 何のAPIを使い、何が取れるのか（3〜5行）
  • ワークフロー全体のキャンバス画像
  • Credentials一覧画面のスクリーンショット ── 「保存したクレデンシャル名」が見える画面（※キーの値は見えなくてOK／むしろ見せない）
  • Executions「Success」画面
  • 出力結果の例 ── AIが最終的に出した文章やデータ（1つ分）
  • 振り返り ── 3〜5行（S3の擬似フローと比べて何が違ったか／どこで詰まったか）

手順

ステップ① 使うAPIを決める

以下から1つ選ぶのがおすすめです。無料かつ登録がシンプルなものを選んでいます。業務に関連するAPIが他にあればそれでもOK。

推奨API（APIキーあり）
  • OpenWeatherMap（https://openweathermap.org/api）：天気予報。動画4と同じ題材。無料登録でキー取得
  • NewsAPI（https://newsapi.org/）：ニュースヘッドライン取得。無料プランあり
  • Qiita API（https://qiita.com/api/v2/docs）：日本語の技術記事検索。トークン取得が簡単
キーなしで動くAPI（簡単優先）
  • Open-Meteo（https://open-meteo.com/）：天気予報。登録不要（認証管理はAI側で実施）
  • 公開RSSフィード：各種ニュースサイトのRSS（認証管理はAI側で実施）
重要：キーなしAPIを選ぶ場合
「認証管理」の採点要素を満たすため、AIノード（Gemini / OpenAI）のAPIキーは必ずCredentialsで管理してください。直接プロンプトやURLに書き込むのは絶対NGです。

ステップ② APIキーを取得する（該当する場合のみ）

上記の推奨APIを使う場合、公式サイトでアカウント登録してAPIキーを発行します。キーは画面に1度だけ表示されることが多いので、安全な場所（パスワードマネージャーなど）にコピーして保管してください。

ステップ③ n8nのCredentialsにキーを登録する

  1. n8n左サイドバーの「Credentials」を開く
  2. 「Add Credential」をクリック
  3. 使うAPIに対応するクレデンシャルタイプを検索（例：OpenAI API、Gemini API、HTTP Header Auth、HTTP Query Auth など）
  4. APIキーを入力し、分かりやすい名前を付けて保存
セキュリティ注意
  • APIキーをHTTP RequestノードのURL・Header・Body に直書きしない
  • 提出するスクリーンショットは、Credentials名は見せてOK／キーの値は映さない
  • キーを誤って公開リポジトリやSlack公開チャンネルに貼らない（漏洩時は即Revokeして再発行）
  • 従量課金系APIの場合は利用上限を設定（OpenAIなら Settings → Billing → Usage limits）

ステップ④ ワークフローを組む

基本構成の例（HTTP Request型）：

  1. Manual Trigger（または Schedule Trigger）
  2. HTTP Request：ステップ①で選んだAPIを叩く。認証はCredentialsから参照する設定にする
  3. AI（Gemini / OpenAI）：取得したデータを要約／整形／判定させる（プロンプトは日本語でOK）
  4. Set（Edit Fields）または出力ノード：最終メッセージを整形（Slackやメール送信は任意）
ヒント：AIに骨組みを書かせて良い
セクション3で学んだ通り、「このAPIから取得してAIで要約するn8nフローのJSONを書いて」とChatGPT/Geminiに依頼すると骨組みが手に入ります。インポートしてから、Credentialsを繋ぐ作業だけ手で行うと早いです。

ステップ⑤ 実行して「Success」にする

  1. 「Test workflow」または Manual Trigger の「Execute step」で実行
  2. HTTP Requestノードをクリックして「Output」タブで実際に取得したAPIデータが見えることを確認
  3. AIノードのOutputで要約結果が出ていることを確認
  4. Executions画面で Success になっていることを確認
