実習課題：定時に動く情報収集・要約・配信エージェントを完成させよう

目的

セクション1〜4で、要件定義・n8nの基本操作・外部API連携・認証管理を学びました。セクション5では、それらを組み合わせて「定時に自動で動き、情報を集め、AIで要約し、配信までする」一連のエージェントを1本完成させます。

ここまでの課題は「部分を動かす」でした。セクション5は「全体を通しで動かす」。動画3で学んだ結合テスト（エンドツーエンドの通しリハーサル）まで実施し、自分の手で作ったエージェントが業務で使えるレベルに達することを体感するのが目的です。

課題

以下の要素をすべて満たす1本のワークフローを作成し、結合テストを通して実行成功まで持っていってください。

  • Schedule Trigger（毎日◯時／毎週◯曜日など、定時起動の設定）
  • 情報収集ノード：RSS Feed Read ノード（推奨）または HTTP Request ノードで外部から情報を取得
  • AI要約ノード：Gemini または OpenAI で取得データを3行以内に要約（件数が多ければ複数件でOK）
  • 配信ノード：Slack／Gmail／Google Sheets／X のいずれか1つに結果を実際に投稿／転記
  • Credentials管理：AI・配信先の認証情報はすべて Credentials 機能で管理（直書きNG）
  • 結合テスト成功：Executionsで「Success」として記録されている

成果物

  • エージェントのコンセプト ── 何の情報を集めてどこに配信するか（3〜5行）
  • ワークフロー全体のキャンバス画像
  • Schedule Trigger の設定画面 ── 実行スケジュール（cron式または頻度選択）が見える画面
  • Executions「Success」画面 ── Manual実行でエンドツーエンドが通った証跡
  • 配信先に実際に投稿／転記された様子のスクリーンショット ── Slackチャンネル／Gmail受信箱／スプシの行／Xの投稿 のいずれか
  • 振り返り ── 3〜5行（結合テストで詰まった点／業務で使えそうな応用案）

手順

ステップ① エージェントのコンセプトを決める

「何の情報を／どれくらいの頻度で／どこに届けるか」を一言で言えるようにします。S1で作った要件定義書の業務をそのまま実装してもOK、S4でAPI連携を練習したテーマを発展させてもOKです。

コンセプトの例
  • 朝のAIニュースブリーフィング：TechCrunch／ITmediaのRSS → Geminiで3行要約 → Slack #ai-daily に毎朝9時投稿
  • 競合調査エージェント：競合3社の公式ブログRSS → 新着記事の要点をAI抽出 → Google Sheets に日付入り行で追記
  • 業界トレンド週報：Qiita API で特定タグの人気記事 → AIで週次傾向コメント → 毎週月曜にGmailで自分宛に送信
  • 採用市場モニタリング：HRテック系RSS → AIで自社事業に関連する記事だけ抽出 → Slack #人事定例 に投稿

ステップ② Schedule Trigger を設定する

  1. 新規ワークフローを作成
  2. 最初のノードに Schedule Trigger を配置
  3. 「Trigger Interval」で起動頻度を設定（例：Every Day at 9:00／Every Monday at 10:00 など）
  4. 保存して、スケジュール設定画面のスクショを撮っておく
テストのコツ
Schedule Triggerが設定時刻に実際に動くのを待つ必要はありません。提出時はManual実行（Test workflow）でエンドツーエンドの動作を確認すればOKです。Schedule設定はあくまで「本番稼働時に自動で動く状態」を作るためのもの。

ステップ③ 情報収集ノードを追加する

推奨は RSS Feed Read ノード（RSSのURLを指定するだけで最新記事一覧を取得）。RSSの代わりに HTTP Request で公開APIを叩くのでも構いません。

使いやすい日本語RSSの例
  • ITmedia NEWS：https://rss.itmedia.co.jp/rss/2.0/news_bursts.xml
  • TechCrunch Japan（英語版は停止していますがグローバル版は有効）：https://techcrunch.com/feed/
  • はてなブックマーク テクノロジー人気：https://b.hatena.ne.jp/hotentry/it.rss
  • 各企業の公式ブログ／プレスリリース（サイトによってRSS提供あり）

ステップ④ AIで3行要約させる

取得した記事群を Gemini または OpenAI に渡して要約します。プロンプトで出力形式を明示するのがコツ。

AIへのプロンプト例

以下の記事タイトルと本文の一部をもとに、3行以内の要約を作ってください。
・1行目：何のニュースか（対象／主語）
・2行目：どう変わったか／何が起きたか
・3行目：なぜ重要か、影響

専門用語は避け、業務で使える敬体で書いてください。

ステップ⑤ 配信先ノードを追加する

以下から1つ選んで実装します。どれも n8n に専用ノードがあり、Credentialsを登録すれば動きます。

  • Slack：指定チャンネルにメッセージ投稿（Slack Appでトークン発行が必要）
  • Gmail：自分宛／チームアドレスにメール送信
  • Google Sheets：指定スプレッドシートに行追加（日付・タイトル・要約を列に並べる形）
  • X (Twitter)：投稿（API従量課金登録が必要／難度高）
配信先を自分だけに絞るのが安全
テスト段階では、Slackなら自分専用チャンネル／Gmailなら自分宛／スプシなら自分のテスト用シートにしておきましょう。誤配信を避けるため、関係者が見るチャンネルやMLには動作確認後に向け先を変えるのが定石です。

ステップ⑥ 結合テストを実施する

部分ではなく通しで動くことを確認します。動画3の「結合テスト」のポイントは以下：

  1. 各ノードを個別に「Execute step」して、出力が想定通りかを1つずつ確認
  2. その後 ワークフロー全体を「Test workflow」で一気通貫実行
  3. Executionsで「Success」になっており、配信先に実際に投稿／転記されていることを確認
  4. エラーで止まったノードがあれば、S6（デバッグ）の知識を先取りしてエラーメッセージを読み、修正して再実行
