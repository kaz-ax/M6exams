実習課題：自分の業務に最適なAIツールを選ぶ ─ 8ツール比較と選定レポート

🔴 提出方法（最重要）

本課題は AI+のGoogleアカウント（株式会社AI School）で作成したGoogleドキュメント で提出します。
共有設定は必ず 「株式会社AI School（リンクを知っているこのグループのメンバーは、誰でも閲覧できます）」 に変更してください。設定画面は以下のようになっていればOKです。

目的

モジュール5ではGPTs／Gems／NotebookLM／Google Workspace Studioを、モジュール6ではn8n／Zapier／Dify／Manusを学びました。それぞれのツールには得意・不得意があり、課題に合わせて最適な武器を選ぶ力が、AIエージェント時代の強力なスキルになります。

この課題の目的は、8ツールを4軸（コスト／難易度／拡張性／得意領域）で比較し、自分の業務に最適なツール（または組み合わせ）を選定する判断力を身につけることです。

課題

自分が自動化したい業務を1つ選び（S1で作った要件定義書をそのまま使ってOK）、以下4つの成果物を1つのGoogleドキュメントにまとめてください。

  1. 対象業務の明確化
  2. 8ツール比較表（4軸）
  3. 選定ツールと理由
  4. ツール組み合わせ提案

成果物

以下の雛形に沿って記入した Googleドキュメント の URL を1つ提出してください。

  • ファイル形式：Googleドキュメント（AI+のGoogleアカウントで作成）
  • 共有設定：株式会社AI School（リンクを知っているこのグループのメンバーは、誰でも閲覧できます）
  • 長さの目安：1〜2ページ（A4相当）

各軸の書き方ガイド

軸 記入するときの観点
コスト 無料／有料プランの金額感、従量課金の有無。例：「無料枠あり（月100タスク）」「月額$20〜」「従量課金（1実行〇〇円）」
難易度 はじめて触る人が動くものを作るまでのハードル。例：「プロンプトだけで動く／低」「ノードを繋ぐ必要／中」「API知識が必要／高」
拡張性 外部サービス連携、分岐・ループ、他ツールとの組み合わせやすさ。例：「Googleアプリ限定／狭い」「API経由で何でも繋がる／広い」
得意領域 そのツールが最も輝く用途。例：「単体Botの設計」「RAGで資料を読み込ませる」「業務フロー全体の自動化」「自律的な調査」

S7の課題は比較・選定力を評価するもので、Zapier／Dify／Manusの実アカウント作成は必須ではありません（S1〜S6で十分ハンズオン経験を積んでいます）。動画で見た情報＋各ツール公式サイトの情報を元に比較すれば足ります。
もちろん、興味があるツールを実際に触って選定理由を書いてもOK（その際は選定理由に「実際に試した結果」を盛り込むとより説得力が増します）。

選定理由の書き方のコツ

  • 自分の業務との接続を明示：「毎朝の情報収集には拡張性よりも手軽さが重要、だからZapier」のように、選んだ軸と業務特性を紐付ける
  • 落選ツールも1つ触れる：「n8nも候補だったが、今回は設定の手軽さを優先してZapierを選んだ」のように比較の過程を見せる
  • コスト・運用も想定：「月間100タスクに収まる見込みなのでZapier無料枠で足りる」など、実運用での試算を含める

組み合わせ提案のヒント

1つのツールだけで解決しようとせず、複数ツールの強みを組み合わせる発想を示します。

  • 例1：Dify で SNS投稿文生成アプリを作成 → n8n のフローから HTTP Request で Dify を呼び出し → Slackへ自動投稿
  • 例2：NotebookLM で社内マニュアルのQ&Aアシスタントを構築 → Gems から参照させて定型業務を実行
  • 例3：Manus で競合3社の週次調査レポートを生成 → n8n で毎週月曜にメール配信
